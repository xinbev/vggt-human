## What SAT Does

### Matcher

SAT-style matcher computes one Hungarian assignment per image/frame over queries and GT people.

Reference behavior:

- Inputs:
  - `pred_confs`: query confidence after sigmoid.
  - `pred_boxes`: normalized `cxcywh` boxes.
  - `pred_j2ds`: projected 2D joints.
  - targets with `boxes`, `j2ds`, `j2ds_mask`.
- Costs:
  - focal-style confidence cost.
  - bbox L1 cost.
  - negative GIoU cost.
  - 2D keypoint cost with visibility mask.
- It solves assignment with `linear_sum_assignment`.

Important point: SAT matching is not just bbox+confidence. It includes 2D joints when available, making matching aware of pose geometry, not only rough object extent.

### Loss

SAT reference loss uses matched indices and computes:

- focal confidence loss, not plain BCE.
- matched L1 for poses/betas/j3ds/j2ds/depths when those outputs/targets exist.
- 2D joints are normalized by image scale and masked by visibility.
- 3D joints are root-aligned before L1.

Important point: SAT does not rely only on direct bbox regression. It uses geometry outputs as supervised signals once available.

### Query/BBox Mechanism

SAT-query reference includes:

- learnable content query + learnable reference bbox query.
- decoder uses reference boxes as positional priors.
- iterative reference refinement in inverse-sigmoid space at each decoder layer.
- bbox heads initialized with final layer zero weights/biases so initial output starts from reference boxes.
- aux bbox outputs from intermediate decoder layers.
- optional denoising queries built from noisy GT boxes/labels/params.

Important point: current VGGT-Omega SMPL path does not implement this SAT query decoder. It appends SMPL query tokens to VGGT aggregator tokens and predicts boxes directly from selected aggregator layers. This is simpler and valid as a first stage, but weaker than SAT's reference-guided query decoding/refinement.

## Assessment: Is Current Scheme Reasonable?

Yes, as a first-stage smoke/bootstrapping scheme.

It is reasonable because:

- The data pipeline is complete for Training bbox supervision.
- The model/loss path trains end-to-end without NaN/OOM.
- Hungarian matching removes slot-order dependency.
- Bbox and GIoU losses are decreasing to meaningful levels.
- SMPL head can predict pose/betas/transl_cam and can be visualized with VGGT camera.

But it is not yet a strong SAT-equivalent training recipe.

Key gaps versus SAT:

1. Confidence uses BCE rather than focal loss.
2. Matching does not use 2D joints or projected geometry.
3. Loss does not supervise projected SMPL joints/bbox through the VGGT camera.
4. There are no aux losses over intermediate decoder/head layers.
5. There is no SAT-style reference bbox query or iterative bbox refinement.
6. There is no denoising query training.
7. There are no training diagnostics for positive vs negative query confidence.
8. There is no quantitative validation metric, only visual inspection.

Main interpretation: current visual issue is not primarily caused by inference post-processing. NMS can make images cleaner, but the training signal itself is not yet strong enough to produce confident, non-duplicate, geometry-consistent query predictions.

## Likely Root Causes

### 1. Confidence Calibration Is Weak

The model's top confidences after 10 epochs are still around 0.25-0.31. For a detector-like query model, positives should be much higher and unmatched queries much lower.

Current BCE confidence loss may be too weak or poorly balanced for 20 queries with about 3 positives/frame.

Need diagnostics:

- matched positive confidence mean/median.
- unmatched negative confidence mean/median.
- positive-negative gap.
- number of predictions above thresholds 0.25/0.3/0.5.

### 2. Duplicate Query Activation

Hungarian matching assigns only one query to each GT, but unmatched queries only receive confidence=0. Their bbox/pose/transl outputs are not directly constrained. If confidence separation remains weak, duplicates appear during inference.

This is expected in DETR-like systems when classification/confidence training is not yet strong.

### 3. BBox Head and SMPL Geometry Are Not Coupled

Current training supervises:

- direct `pred_boxes` vs GT bbox.
- direct `pred_transl_cam` vs GT translation.
- direct pose/betas vs GT.

But it does not enforce:

`pred_pose + pred_betas + pred_transl_cam + VGGT camera -> projected joints/vertices/bbox` matches the image-space target.

Therefore a query can have a bbox that looks partly reasonable while SMPL joints/translation are not geometrically consistent enough.

### 4. Matching Uses Only Boxes, Not Pose Geometry

SAT matcher includes 2D keypoint cost. Current matcher has `cost_kpts=0.0` and no `pred_j2ds` output. This can make matching ambiguous when boxes overlap or are close.

### 5. Frozen Aggregator Limits Adaptation

Frozen aggregator is good for first-stage stability. But after 10 epochs, if query specialization stalls, later training may need to unfreeze a small top portion of aggregator or add a lightweight query decoder/refinement layer.

Do not unfreeze widely before improving losses/diagnostics.

## Recommended Next Direction

Do not focus first on prettier visualization. Focus on making training signals and metrics stronger.

Priority order:

1. Add diagnostics/metrics to understand convergence.
2. Switch confidence loss to focal loss and log positive/negative confidence separation.

## Implementation Plan

### Step 1: Training Diagnostics

Modify `HungarianSMPLLoss` to return additional scalar metrics without affecting gradients:

- `metric_num_targets`
- `metric_num_matched`
- `metric_conf_pos_mean`
- `metric_conf_neg_mean`
- `metric_conf_gap`
- `metric_pred_count_025`
- `metric_pred_count_030`
- `metric_pred_count_050`
- `metric_bbox_iou_mean` for matched direct boxes.
- `metric_projected_bbox_iou_mean` once projected bbox exists.

These should be logged by existing `format_log` because it prints all finite scalars.

Success criteria after continued training:

- `metric_conf_pos_mean` rises substantially above negatives.
- `metric_conf_neg_mean` falls below about 0.1-0.15.
- `metric_conf_gap` becomes positive and grows.
- predictions above 0.3 approach average person count rather than 8-10/query duplicates.

### Step 2: Focal Confidence Loss

Replace BCE in `HungarianSMPLLoss` with SAT-style focal loss or make it configurable:

Config fields:

```yaml
loss:
  conf_loss_type: focal
  conf_focal_alpha: 0.25
  conf_focal_gamma: 2.0
  conf_weight: 2.0
```

Rationale:

- SAT reference uses focal confidence loss.
- Current confidence remains low and clustered.
- Focal loss should help push hard positives/negatives apart.

Keep compatibility with BCE via config if needed.

### Step 3: Projected SMPL BBox Loss With VGGT Camera

Add optional training loss:

```yaml
model:
  enable_camera: true   # needed only when using projected VGGT camera loss

loss:
  projected_bbox_weight: 1.0
  projected_giou_weight: 1.0
  use_vggt_camera_projection: true
```

Training computation:

1. Use matched queries from Hungarian indices.
2. Decode SMPL from `pred_poses` or `pred_pose_6d -> axis-angle`, `pred_betas` using project-local `SMPLLayer`.
3. Add `pred_transl_cam` to vertices or joints.
4. Decode VGGT `pose_enc` using `encoding_to_camera(..., image_size_hw=(518, 518))` to get intrinsics.
5. Project vertices or robust selected joints to 2D.
6. Build normalized projected bbox in `cxcywh` over 518 input coordinates.
7. Compare to matched `gt_boxes` using L1 + GIoU.

Use vertices for tighter bbox if memory permits; otherwise use selected body joints as a first pass.

Important engineering constraints:

- `smplx` and SMPL assets are optional dependencies; only required when projected loss is enabled.
- Baseline import must not require `smplx`.
- Avoid decoding all 20 queries if possible; decode only matched queries to control memory.
- If `enable_camera=true`, load baseline camera head weights. The training checkpoint currently does not include camera head because prior model disabled it. Future training with this loss must build model with `enable_camera=true` and load baseline first.
- Keep `camera_head` frozen initially.

Success criteria:

- `loss_projected_bbox` decreases.
- visualized projected joints/bbox become better aligned.
- duplicate query confidence decreases via stronger geometry consistency.

### Step 4: Add 2D Joint Targets If Available/Generated

Current preprocessing by `--projection-source vertices` likely can generate projected boxes. Extend preprocessing output to include projected SMPL joints if not already present:

- `j2ds`: projected SMPL joints in original image pixels or 518 input pixels.
- `j2ds_mask`: visibility/in-image mask.
- clearly record coordinate space.

Update `BedlamDataset` to load:

- `gt_j2ds`: `[S, M, J, 2]`
- `j2ds_mask`: `[S, M, J, 2]` or `[S, M, J, 1]`

Update `flatten_smpl_targets` to include `j2ds` and `j2ds_mask`.

Then enable matcher keypoint cost:

```yaml
matching:
  cost_kpts: 5.0  # tune from 2.0-10.0
```

And add projected joints loss:

```yaml
loss:
  projected_j2d_weight: 1.0
```

This moves closer to SAT reference: matching uses bbox + GIoU + 2D keypoints.

### Step 5: Aux Losses From Intermediate SMPL Layers

Current `SMPLRegressionHead` already has `return_aux`, but config/model wiring may not enable or train aux outputs.

Plan:

- Add model config `smpl_return_aux: true`.
- Wire to `AggregatorSMPLHead(return_aux=True)`.
- Compute losses on aux outputs with smaller weights, e.g. 0.5 for intermediate layers.

SAT/DETR-like models usually benefit from aux losses because they stabilize early decoder/head layers.

Do this after confidence/projected bbox diagnostics are implemented.

### Step 6: Continue Training Plan

#### Stage 2A: Stronger Head Training, Aggregator Frozen

Use current checkpoint as resume.

Suggested config changes:

```yaml
model:
  enable_camera: true
  freeze_aggregator: true
  train_smpl_query_token: true
  predict_boxes: true
  predict_id_embed: false  # optional, since id_weight=0.0

  conf_focal_alpha: 0.25
  conf_focal_gamma: 2.0

  giou_weight: 2.0
  pose_weight: 1.0

  transl_cam_weight: 0.1
  projected_bbox_weight: 1.0
  projected_giou_weight: 1.0
  id_weight: 0.0

  cost_conf: 1.0
  cost_bbox: 5.0
  cost_giou: 2.0
  cost_kpts: 0.0 initially

optim:
  lr: 5e-5
  epochs: 5 to 10 additional epochs
```

Training duration:

- Resume from `checkpoint_latest.pt`.
- Run 5 epochs first.
- Evaluate diagnostics and visualization on a fixed 20-50 image subset.
- Continue another 5 epochs only if metrics improve.

Stop/continue criteria:

Continue if:

- `metric_conf_gap` improves.
- `metric_conf_neg_mean` decreases.
- `metric_conf_pos_mean` increases or stays above negatives.
- `loss_projected_bbox` decreases.
- matched direct bbox IoU improves.

Stop and revise if after 2 epochs:

- confidence pos/neg means remain almost equal.
- `metric_pred_count_030` remains far above GT count.
- projected bbox loss does not decrease.
- training becomes unstable/NaN/OOM.

#### Stage 2B: Add 2D Keypoint Matching/Loss

Only after preprocessing/dataset provides reliable `j2ds`.

Suggested settings:

```yaml
matching:
  cost_kpts: 5.0

loss:
  projected_j2d_weight: 1.0
```

Run 5 additional epochs frozen aggregator.

#### Stage 2C: Partial Aggregator Unfreeze

Only if Stage 2A/2B saturates but metrics remain insufficient.

Minimal unfreeze strategy:

- keep most aggregator frozen.
- unfreeze last 2-4 aggregator blocks and SMPL query token/head.
- use lower LR for aggregator, e.g. `1e-5`.
- SMPL head LR `5e-5`.

This requires optimizer parameter groups.

Do not unfreeze all aggregator immediately because dataset is relatively small (7619 frames) and current validation is visual/Training-only.

### Step 7: Evaluation Protocol

Since no Test split exists, create a fixed Training diagnostic subset that is not used for checkpoint selection as much as possible.

Recommended:

- Select 100-200 fixed frames across sequences.
- Save list under `outputs/eval/smpl_train_subset_frames.txt` or config-controlled path.
- Run inference every epoch or every N epochs.
- Compute metrics against sidecar GT boxes:
  - AP-like or simple precision/recall at IoU 0.5 using confidence thresholds.
  - mean matched IoU after Hungarian on predicted vs GT boxes.
  - duplicate rate: predictions per GT after NMS/threshold.
  - GT recall@topK.
  - projected bbox IoU if SMPL projection enabled.

Do not rely only on one visualization image.

Useful metrics:

- `box_recall_iou50_top20`
- `box_precision_iou50_conf030`
- `mean_best_gt_iou`
- `duplicate_predictions_per_gt`
- `conf_pos_neg_gap`
- `projected_bbox_iou_mean`

## Is the Current Effect Mainly Training Insufficient?

Partly, but not only.

Evidence for training insufficiency:

- confidence values are low and clustered even after 10 epochs.
- many queries remain above 0.25 but none are strongly confident.
- duplicate predictions persist.

Evidence that loss design is also insufficient:

- current loss lacks SAT-style focal confidence.
- matching lacks 2D keypoint/geometry costs.
- no projected SMPL bbox/joints loss through VGGT camera.
- no aux losses or reference refinement.

Therefore, simply training the same configuration much longer may help a little but is unlikely to fully solve confidence separation and geometry consistency. Continue training should be paired with diagnostics and at least focal confidence + projected bbox loss.

## Risks

- Enabling camera head during training requires loading baseline camera weights and freezing camera head unless explicitly training camera. Do not let camera head randomly initialize.
- Projected SMPL loss requires `smplx` and SMPL assets. Keep lazy dependency and clear error.
- Projecting vertices for all matches may add memory cost; start with matched queries only.
- Current data has only Training. Use fixed subset metrics to reduce overfitting blindness.
- If BEDLAM `smplx_transl` is not exactly in the same camera convention as VGGT projected camera, projected loss may conflict. Need verify on GT side: project GT SMPL with VGGT/known intrinsics and compare sidecar boxes before training the loss.

## Final Recommendation

Current scheme is a valid first-stage baseline, but the observed visualization issue should be treated as a training/loss/metrics issue rather than a cosmetic visualization issue.

Next step should not be only NMS or more epochs. NMS is useful for reading visualizations, but it does not improve model ability.

The highest-value next move is:

1. Add confidence diagnostics.
2. Replace BCE confidence with focal loss.
3. Add VGGT-camera projected SMPL bbox loss.
4. Resume training for 5 epochs with frozen aggregator and lower LR.
5. Decide whether to continue based on confidence separation and projected bbox/box IoU metrics.
