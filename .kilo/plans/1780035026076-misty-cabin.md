# Plan: SMPL ROI-Pooled Human Query + 3D-First Training + PLY Visualization

## Goal
Implement a new experimental SMPL refinement branch where each human query is initialized as:

```text
human_query = learnable token + box prior embedding + bbox region patch-token pooling embedding
```

Then train with stronger 3D SMPL supervision, keep 2D projected loss as auxiliary, and extend visualization to export predicted and GT SMPL 3D point clouds as `.ply` files.

## Current Context
- Main code is in `vggt_omega/`; `.reference/` remains read-only.
- Current query path:
  - `Aggregator.forward()` builds `smpl_query_token`.
  - `_build_smpl_reference_boxes()` produces reference boxes from GT/noisy/fallback boxes.
  - `_add_smpl_box_prior()` adds `MLP([cx, cy, w, h, valid])` to query tokens.
  - `patch_tokens = self.patch_embed(images)` is currently computed after the box-prior query injection.
  - `tokens = [camera, register, smpl_query, patch_tokens]` then enters frame/inter-frame attention.
- Current SMPL joint loss already exists in `HungarianSMPLLoss`:
  - `loss_joints3d`
  - `loss_projected_joints2d`
  - projected bbox losses
- Current visualization already draws predicted and GT projected 2D joints, but does not export 3D PLY.

## Implementation Steps

### 1. Add Optional ROI Patch Pooling To Aggregator
Modify `vggt_omega/models/aggregator.py`.

Add new `Aggregator.__init__` args with baseline-safe defaults:

```python
smpl_query_patch_pool: bool = False
smpl_query_patch_pool_expand: float = 0.10
```

When enabled and `num_smpl_queries > 0`, add a trainable adapter:

```python
self.smpl_patch_pool_embed = nn.Sequential(
    nn.Linear(embed_dim * 2 + 6, embed_dim),
    nn.GELU(),
    nn.LayerNorm(embed_dim),
    nn.Linear(embed_dim, embed_dim),
)
```

Use `embed_dim * 2 + 6` because the pooled feature should include:
- mean pooled patch token inside box
- max pooled patch token inside box
- box metadata `[cx, cy, w, h, valid, area]`

Move patch embedding before final query composition:

```text
1. build learnable smpl query
2. build reference boxes
3. patch_tokens = patch_embed(images)
4. add box prior embedding
5. add ROI patch pooling embedding
6. concat tokens
```

Implement helper methods:

```python
_add_smpl_patch_pool_prior(...)
_pool_box_patch_tokens(...)
_patch_box_mask(...)
```

Pooling design:
- `patch_tokens` shape is `(B*S, P, C)`.
- Convert `smpl_reference_boxes` from `(B,S,Q,4)` to flattened `(B*S,Q,4)`.
- Patch grid is `(height // patch_size, width // patch_size)`.
- Build normalized patch centers in `[0,1]` using cell centers.
- Expand box width/height by `smpl_query_patch_pool_expand` to tolerate noisy priors.
- For each query, select patch tokens whose centers fall inside the expanded box.
- Mean pool with mask; max pool with masked fill.
- If a box is invalid or has no selected patches, pooled feature should become zero and metadata valid flag should be zero.
- Return `smpl_query_token + pool_embed`.

Important baseline preservation:
- Default `smpl_query_patch_pool=False`, so existing configs/checkpoints keep behavior.
- Existing `smpl_query_box_prior=True` remains independent.
- Fallback boxes can still pool if no GT boxes are passed, but valid mask should reflect whether true prior exists; use the mask metadata so the adapter can learn distrust fallback/no-prior features.

### 3. New 3D-First Experiment Config
Create `configs/train_smpl_dab_roi_pool_3d_refine.yaml` based on `train_smpl_dab_joint_refine.yaml`.

Recommended config changes:

```yaml
experiment:
  name: smpl_dab_roi_pool_3d_refine_20q
  output_dir: outputs/train/smpl_dab_roi_pool_3d_refine_20q

model:
  smpl_query_box_prior: true
  smpl_query_patch_pool: true
  smpl_query_patch_pool_expand: 0.12
  train_smpl_query_token: true
  train_smpl_box_prior_embed: true
  train_smpl_patch_pool_embed: true
  freeze_aggregator: true
  freeze_camera_head: true
  freeze_aggregator_forward: false

checkpoint:
  resume: outputs/train/smpl_dab_joint_refine_20q/checkpoint_latest.pt
  resume_optimizer: false
  resume_strict: false

optim:
  epochs: 100
  lr: 0.00001
  batch_size: 1

training_prior:
  center_noise: 0.04
  size_noise: 0.10
  drop_prob: 0.03

loss:
  pose_weight: 2.0
  betas_weight: 0.2
  transl_cam_weight: 0.8
  joints3d_weight: 6.0
  projected_joints2d_weight: 0.35
  projected_bbox_weight: 0.10
  projected_giou_weight: 0.10
  bbox_weight: 3.0
  giou_weight: 1.5
  conf_weight: 1.0
  aux_weight: 0.15
```

Rationale:
- Main objective is good SMPL, so 3D joints dominate.
- 2D projected joints remain auxiliary, not primary.
- Projected bbox weight is reduced so box silhouette does not overrule body geometry.
- bbox/GIoU remain nonzero to preserve person binding.
- Resume from the latest joint-refine checkpoint because it already has joint supervision and stable DAB binding.
- Set total epochs to 100 so if checkpoint epoch is 60, this runs 40 more epochs.

### 5. Extend Visualization To Export PLY
Modify `scripts/vis/visualize_smpl_inference.py`.

Add args:

```python
--export-ply
--ply-top-k int default=3
--ply-use-vertices  # optional; default joints-only or both as described below
```

Recommended export behavior:
- When `--export-ply` is set and SMPL model dir is provided:
  - Decode predicted SMPL vertices and joints for selected predictions.
  - Decode GT SMPL vertices and joints for the same `query_index` where GT exists.
  - Save separate files per prediction/query:
    - `{stem}_rank{rank:02d}_q{query}_pred_joints.ply`
    - `{stem}_rank{rank:02d}_q{query}_gt_joints.ply`
    - `{stem}_rank{rank:02d}_q{query}_pred_vertices.ply`
    - `{stem}_rank{rank:02d}_q{query}_gt_vertices.ply`
  - Also save one combined comparison PLY per query:
    - `{stem}_rank{rank:02d}_q{query}_pred_gt_joints.ply`
- Use camera-space points by adding `transl_cam` to SMPL outputs.
- Color conventions:
  - predicted joints/vertices: red-ish `(255, 64, 64)`
  - GT joints/vertices: white or green `(64, 255, 128)`
- For joints PLY, write point cloud only.
- For vertices PLY, write point cloud only first; do not require faces. This avoids face format complexity and makes it easy to inspect in MeshLab/CloudCompare.

Implementation helpers:

```python
write_ply_points(path: Path, points: np.ndarray, colors: np.ndarray | tuple[int,int,int])
export_prediction_gt_ply(...)
```

Also add 3D point arrays to JSON for selected results only if useful; avoid dumping full vertices into JSON because huge files are unnecessary.

## Risks And Mitigations
- ROI pooling can inject wrong appearance if prior box is noisy or dropped.
  - Mitigation: expand boxes moderately, include valid/area metadata, keep noisy-prior training.
- Mean pooling may be too coarse for pose.
  - Mitigation: use mean + max pooling and box metadata; keep implementation minimal before adding grid ROI pooling.
- Freezing Aggregator could freeze the new adapter by accident.
  - Mitigation: explicitly unfreeze `smpl_patch_pool_embed` and optionally `smpl_box_prior_embed` under config flags.
- PLY output can become large if vertices are exported for many predictions.
  - Mitigation: use `--ply-top-k`, default 3, and point-only PLY.
