## Goal

- Default maximum people / queries becomes 20.
- Preprocess missing or inconsistent bbox annotations into a project-local output directory, without mutating original BEDLAM data.
- Train with per-frame Hungarian query-to-person matching using bbox + confidence costs and matched SMPL losses.
- Add the necessary model outputs (`pred_boxes`, optional `pred_id_embed`) for matching and later identity consistency.
- Add a shell launcher matching the style of `.reference/stage2_body_j2d1e-4.sh`.

This plan only adapts reference ideas; it will not import `.reference/` at runtime.

## Important Constraint: BBox Generation

A bbox cannot be reliably generated from only `smplx_root_pose`, `smplx_body_pose`, `smplx_shape`, and `smplx_transl` unless one of these is available:

1. Existing bbox fields in person dict.
2. Existing 2D joints/keypoints in person dict.
3. SMPL/SMPL-X body model + camera intrinsics for projection.

The preprocessing implementation will support all three, in this priority order:

1. Use and repair existing bbox fields if present.
2. Generate bbox from available 2D joints/keypoints if present.
3. Optionally generate 2D joints and bbox using SMPL projection when `--use-smpl-projection` is enabled and `smplx` + model assets are available.
4. If none is possible, mark the person invalid for bbox matching and include a clear summary count. With `--require-boxes`, preprocessing fails instead of silently producing unusable training targets.

### 3. `vggt_omega/training/hungarian_losses.py`

Purpose: matched loss for query-based SMPL training.

Class:

```python
class HungarianSMPLLoss(nn.Module):
    def __init__(
        matcher: HungarianSMPLMatcher,
        pose_weight: float = 1.0,
        betas_weight: float = 0.1,
        cam_weight: float = 0.1,
        conf_weight: float = 1.0,
        bbox_weight: float = 5.0,
        giou_weight: float = 2.0,
        id_weight: float = 0.0,
        id_temperature: float = 0.07,
    ) -> None
```

Inputs:

- `predictions` with:
  - `pred_confs`: `[B, S, Q, 1]`
  - `pred_boxes`: `[B, S, Q, 4]`
  - `pred_pose_6d`: `[B, S, Q, 144]`
  - `pred_betas`: `[B, S, Q, 10]`
  - `pred_cam`: `[B, S, Q, 3]`
  - optional `pred_id_embed`: `[B, S, Q, D]`
- `batch` with padded tensors and/or target list.

Losses:

- `loss_conf`: BCE/focal-style query confidence. Matched queries target 1, unmatched target 0.
- `loss_bbox`: matched L1 bbox loss.
- `loss_giou`: matched generalized IoU loss.
- `loss_pose`: matched L1 on pose 6D.
- `loss_betas`: matched L1 on betas with existing betas dimension weights.
- `loss_cam`: matched L1 on camera translation.
- `loss_id`: optional supervised contrastive identity loss when valid `person_ids` exist in the batch.
- `loss_total` weighted sum.

Important: Hungarian matching is per-frame over `B*S` items. Cross-frame identity consistency is not forced if dataset has no stable `person_id`; in that case `loss_id` is skipped or zero.

## Identity Consistency Scope

Hungarian matching solves per-frame query/person assignment. It does not by itself guarantee cross-frame identity. To avoid overclaiming:

- The implementation will preserve `person_ids` from preprocessing if present.
- The model can output `pred_id_embed`.
- `HungarianSMPLLoss` can include optional `loss_id` only when valid person IDs exist.
- Default `id_weight=0.0`, so initial implementation focuses on robust frame-level matching.
- If the preprocessed data lacks stable cross-frame person IDs, true cross-frame ID consistency cannot be supervised. It can be added later once IDs are available.

set -euo pipefail

REPO_ROOT="/home/zhw/lab_users/xyb/home/projects/vggt-omega"

## Risks / User-Provided Data Requirements

- If original pkl lacks bbox and 2D joints, preprocessing requires SMPL projection mode with valid `smplx`/SMPL assets and camera intrinsics.
- If data has no stable `person_id`, cross-frame identity loss cannot be supervised; Hungarian only solves frame-level query assignment.
- Current `VGGTOmega.forward` uses CUDA autocast unconditionally; server should use CUDA. CPU local full forward may still fail independently.
- Freezing aggregator while keeping `smpl_query_token` trainable requires careful parameter filtering because `smpl_query_token` lives inside aggregator.
