# SMPL Matching and Regression Implementation Plan

## Scope and Reference Boundary

This plan adapts SAT-HMR SMPL matching/regression ideas into the main project under `vggt_human/`. It will not import anything from `.reference/` at runtime.

References inspected:

- `.reference/SAT-HMR-smpl/training_optional/matcher_reference.py`
- `.reference/SAT-HMR-smpl/heads/smpl_regression_head.py`
- `.reference/SAT-HMR-smpl/geometry/rotation_conversions.py`
- `.reference/SAT-HMR-smpl/training_optional/losses_reference.py`
- `docs/reference-map.md`
- current `vggt_human/models/aggregator.py`, `vggt_human/models/vggt_omega.py`, and current heads

Reference interpretation:

- Matcher: adapt the Hungarian matching cost structure from SAT-HMR for multi-query SMPL/person predictions.
- Regression head: adapt SAT-HMR residual iterative regression from mean SMPL parameters over four cached aggregator layers.
- Rotation conversion: adapt 6D rotation to axis-angle utilities into the main project.
- Baseline: preserve existing camera/depth behavior by default. SMPL functionality is enabled only through new constructor/config flags.

## Current Code Facts

Current aggregator token order is:

```text
[camera, registers, image_patches]
```

Current important tensor shapes:

```text
images: (B, S, 3, H, W)
raw aggregator token dim: C = embed_dim, default 1024
cached aggregator token dim: 2C, default 2048
aggregated_tokens_list[layer_idx]: (B, S, T, 2C) or None
cached layers: 4, 11, 17, 23
patch_token_start: 1 + num_register_tokens
```

Current branch use:

- Camera head uses `tokens[:, :, :patch_token_start]`, currently camera + registers.
- Dense head uses `tokens[:, :, patch_token_start:]`, currently image patches.
- Text alignment also uses `tokens[:, :, :patch_token_start]`.

SMPL integration must not let camera/depth accidentally consume SMPL query tokens.

## Target Token Routing

After implementation, optional SMPL-enabled aggregator token order will be:

```text
[camera, registers, smpl_queries, image_patches]
```

Token ranges will be explicit rather than relying on a single ambiguous `patch_token_start`:

```text
camera:       [0, 1)
registers:    [1, 1 + R)
smpl_queries: [1 + R, 1 + R + Q)
patches:      [1 + R + Q, T)
```

Routing rules:

- Camera branch receives only `[camera, registers]` and keeps existing logic.
- Depth branch receives only `[image_patches]` and keeps existing logic.
- SMPL branch receives only `[smpl_queries]` from the four cached aggregator layers.
- Register-type inter-frame attention treats `[camera, registers, smpl_queries]` as context tokens and leaves image patches in the patch segment, preserving the original sparse cross-frame behavior for patches.

When `num_smpl_queries=0`, the layout degenerates exactly to the current baseline:

```text
[camera, registers, image_patches]
patch_start = 1 + R
camera_register_end = 1 + R
```

- Keeps routing explicit and testable.

### 2. `vggt_human/models/heads/smpl_head.py`

Add the SMPL regression adapter and SAT-HMR-style residual head.

Components:

- `MLP`
- `_get_clones`
- `SMPLRegressionHead`
- `AggregatorSMPLHead`

`SMPLRegressionHead` input:

```text
hidden_states: (L, BQBatch, Q, C)
```

Where `L=4`, `BQBatch=B*S`, `Q=num_smpl_queries`, `C=2*embed_dim`.

`SMPLRegressionHead` output:

```text
pred_poses: (BQBatch, Q, 72)       # axis-angle, 24*3
pred_pose_6d: (BQBatch, Q, 144)    # useful for training/debug
pred_betas: (BQBatch, Q, 10)
pred_confs: (BQBatch, Q, 1)
pred_cam: (BQBatch, Q, 3)
aux_outputs optional with stacked per-layer predictions
```

`AggregatorSMPLHead` input:

```text
aggregated_tokens_list: list[Tensor | None]
token_layout: AggregatorTokenLayout
```

It extracts cached SMPL query tokens:

```text
for layer in [4, 11, 17, 23]:
    x = aggregated_tokens_list[layer][:, :, smpl_start:smpl_end, :]
    x -> (B*S, Q, C)
hidden_states -> (4, B*S, Q, C)
```

It calls `SMPLRegressionHead`, then reshapes outputs back to frame-aware layout:

```text
pred_poses: (B, S, Q, 72)
pred_pose_6d: (B, S, Q, 144)
pred_betas: (B, S, Q, 10)
pred_confs: (B, S, Q, 1)
pred_cam: (B, S, Q, 3)
```

Mean parameter handling:

- Accept optional `mean_pose_6d` and `mean_shape` tensors in constructor.
- If not provided, initialize mean pose as identity 6D rotation for 24 joints and mean shape as zeros.
- Do not read local absolute paths or require SMPL assets.

Residual iteration:

```text
pose_6d = mean_pose_6d
shape = mean_shape
for each of the four cached layers:
    pose_6d = pose_6d + pose_head[layer](hidden[layer])
    shape = shape + shape_head[layer](hidden[layer])
    pred_cam = cam_head(hidden[layer])
    pred_confs = sigmoid(conf_head(hidden[layer]))
    pred_poses = rot6d_to_axis_angle(pose_6d)
```

This aligns with SAT-HMR's mean-parameter residual refinement design.

### 4. `vggt_human/training/smpl_matcher.py`

Add a project-local Hungarian matcher adapted from `matcher_reference.py`.

Public API:

```python
class HungarianSMPLMatcher(nn.Module):
    def __init__(
        cost_conf: float = 1.0,
        cost_bbox: float = 1.0,
        cost_giou: float = 1.0,
        cost_kpts: float = 10.0,
        j2ds_norm_scale: float = 518.0,
        require_boxes: bool = True,
        require_j2ds: bool = True,
    ) -> None
```

Expected output fields:

```text
outputs["pred_confs"]: (B, Q, 1) or (B, S, Q, 1)
outputs["pred_boxes"]: (B, Q, 4) or (B, S, Q, 4), normalized cxcywh if bbox costs enabled
outputs["pred_j2ds"]:  (B, Q, J, 2) or (B, S, Q, J, 2), pixel units unless caller changes j2ds_norm_scale
```

Expected target fields per batch item:

```text
target["boxes"]: (N, 4), normalized cxcywh if bbox costs enabled
target["j2ds"]: (N, J, 2) if keypoint cost enabled
target["j2ds_mask"]: (N, J, 2) if keypoint cost enabled
```

Frame handling:

- Matcher itself operates on flattened batch items `(Bmatch, Q, ...)`.
- Add a helper that flattens `(B, S, ...)` into `(B*S, ...)` when targets are also provided per frame.
- Do not guess dataset target frame structure in model code.

Dependency handling:

- Current `scipy` is only optional under `[project.optional-dependencies].demo`.
- To avoid breaking baseline import, import `linear_sum_assignment` lazily inside `forward` and raise a clear error if `scipy` is missing.
- Do not add `scipy` to required dependencies unless the user later requests training dependency changes.

Edge cases:

- Empty targets return empty source/target index tensors for that batch item.
- All matching costs disabled raises `ValueError`.
- If a required output/target key is missing for an enabled cost, raise `ValueError` with the missing key name.


- Add:


- Replace `self.patch_token_start = 1 + num_register_tokens` with explicit layout:

```python
self.token_layout = AggregatorTokenLayout(
    camera_start=0,
    camera_end=1,
    register_start=1,
    register_end=1 + num_register_tokens,
    smpl_start=1 + num_register_tokens,
    smpl_end=1 + num_register_tokens + num_smpl_queries,
    patch_start=1 + num_register_tokens + num_smpl_queries,

```

Keeping `self.patch_token_start` as an alias preserves minimal compatibility.

- In `init_weights`, initialize SMPL query token when present.

```python
tokens = torch.cat([camera_token, register_token, smpl_query_token, patch_tokens], dim=1)
```

if `num_smpl_queries > 0`; otherwise keep current concatenation.

- Return:

```python

```

instead of `(outputs, patch_token_start)`.

- In `_run_inter_frame_attention_block`, replace local `patch_token_start` use with `self.token_layout.patch_start`. This means register-type inter-frame attention updates camera/register/SMPL-query context tokens together and leaves image patches untouched.

Why minimal:

- No change to frame/global attention mechanics.
- No change to image patch embedding.
- No change to cached layer indices.
- Baseline layout remains identical when `num_smpl_queries=0`.

## SMPL Matching Design Details

The matcher will be used during training only. It will not be called from `VGGTOmega.forward`.

Cost terms adapted from SAT-HMR:

```text
confidence focal-style positive cost:
    alpha * (1 - pred_conf)^gamma * -log(pred_conf)

bbox L1 cost:
    cdist(pred_boxes, target_boxes, p=1)

gIoU cost:
    -generalized_box_iou(cxcywh_to_xyxy(pred_boxes), cxcywh_to_xyxy(target_boxes))

2D keypoint cost:
    mean visible L1 distance over first 22 joints, normalized by j2ds_norm_scale
```

Output:

```text
indices: list[(src_indices, target_indices)] length Bmatch
```

These indices can feed a later SMPL loss module, or the existing SAT-HMR-style loss adapter if added in a later task.

Important target assumptions to document in code docstring:

- `boxes` are normalized `cxcywh`.
- `j2ds` are in pixel coordinates unless `j2ds_norm_scale` is set for another convention.
- `j2ds_mask` is broadcast-compatible with flattened `(J*2)` keypoint coordinates.
- The matcher works on per-frame predictions after caller flattens `(B, S)` if needed.

## SMPL Regression Data Flow

End-to-end with SMPL enabled:

```text
images
  -> Aggregator
     tokens per frame: [camera, registers, smpl_queries, image_patches]
     cached layers: 4, 11, 17, 23
  -> camera head
     consumes [camera, registers]
     outputs pose_enc as before
  -> dense head
     consumes image patches starting at token_layout.patch_start
     outputs depth/depth_conf as before
  -> SMPL head
     consumes smpl query slice from all four cached layers
     hidden_states: (4, B*S, Q, 2048)
     residual iterative mean-SMPL refinement
     outputs frame-aware SMPL predictions
```

SMPL output shapes from `VGGTOmega.forward`:

```text
pred_poses: (B, S, Q, 72)
pred_pose_6d: (B, S, Q, 144)
pred_betas: (B, S, Q, 10)
pred_confs: (B, S, Q, 1)
pred_cam: (B, S, Q, 3)
```

If `return_aux=True` is supported in `AggregatorSMPLHead`, auxiliary outputs will be:

```text
aux_outputs["pred_poses"]: (4, B, S, Q, 72)
aux_outputs["pred_pose_6d"]: (4, B, S, Q, 144)
aux_outputs["pred_betas"]: (4, B, S, Q, 10)
aux_outputs["pred_confs"]: (4, B, S, Q, 1)
aux_outputs["pred_cam"]: (4, B, S, Q, 3)
```

## Implementation Order

1. Add `AggregatorTokenLayout`.
2. Extend aggregator with optional SMPL query token and explicit token layout.
3. Update camera/text heads to use camera/register range from token layout.
4. Wire `VGGTOmega` to route token layout and optional SMPL head.
5. Extend rotation utilities with 6D-to-axis-angle conversion.
6. Add `SMPLRegressionHead` and `AggregatorSMPLHead`.
7. Export SMPL heads from `heads/__init__.py`.
8. Add training matcher and export it from `vggt_human/training/__init__.py`.
9. Run the validation checks above.
