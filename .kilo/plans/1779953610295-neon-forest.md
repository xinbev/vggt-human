# Experiment A Next Plan: Confidence Calibration + Duplicate Suppression

## Goal

Continue the VGGT-native SMPL query head route instead of migrating SAT-HMR wholesale.

The immediate goal is to fix the current failure mode:

- Stage 2 projected bbox reduced duplicates but boxes remained inaccurate.
- BBox-refine training improved top-20 recall / mean best IoU slightly, but pushed too many queries above confidence threshold:
  - `num_pred_conf`: 417 -> 7546
  - `duplicate_predictions_per_gt`: 0.0166 -> 0.3956

Do not add 2D joint loss yet. Current problem is still box/conf/query specialization, not final pose accuracy.

## Task 1: Quality-Aware Confidence Target

### File

`vggt_omega/training/hungarian_losses.py`

### Current Behavior

Confidence target is hard binary:

```python
conf_target = torch.zeros_like(pred_confs)
for frame_idx, (src_idx, _) in enumerate(indices):
    conf_target[frame_idx, src_idx, 0] = 1.0
loss_conf = focal_or_bce(pred_confs, conf_target)
```

This makes confidence represent matched/unmatched assignment, not box quality.

### New Config Fields

Add to `HungarianSMPLLoss.__init__`:

```yaml
loss:
  conf_target_type: binary        # binary | matched_iou
  conf_iou_min: 0.0               # lower clamp for matched IoU target
  conf_iou_power: 1.0             # optional shaping, target = iou ** power
```

Defaults must preserve current behavior:

```python
conf_target_type="binary"
conf_iou_min=0.0
conf_iou_power=1.0
```

### New Behavior

If `conf_target_type == "binary"`:

- keep current behavior.

If `conf_target_type == "matched_iou"`:

- for each matched query, compute direct bbox IoU between `pred_boxes[frame_idx, src_idx]` and matched target boxes.
- detach the IoU.
- set positive target to `clamp(iou, min=conf_iou_min) ** conf_iou_power`.
- unmatched queries remain 0.

This makes confidence learn quality:

```text
higher confidence only if matched direct bbox is high quality
```

### Metrics To Add

Log extra scalar metrics:

- `metric_conf_target_pos_mean`
- `metric_conf_target_pos_min`
- `metric_conf_target_pos_max`

These should be detached scalars and printed by existing `format_log`.

### Notes

Compute matched IoU once and reuse for:

- `metric_bbox_iou_mean`
- quality-aware confidence target

Avoid backprop through IoU target for confidence target by detaching.

## Task 2: Duplicate Unmatched Confidence Suppression

### File

`vggt_omega/training/hungarian_losses.py`

### Motivation

Hungarian assigns one query per GT. Unmatched duplicate queries can still predict plausible boxes. If confidence loss is insufficient, they survive thresholding.

### New Config Fields

```yaml
loss:
  duplicate_conf_weight: 0.0
  duplicate_iou_threshold: 0.5
```

Defaults preserve current behavior.

### Behavior

For each frame:

1. Identify unmatched query indices.
2. Compute IoU between each unmatched `pred_box` and all GT boxes.
3. `duplicate_mask = max_iou > duplicate_iou_threshold`.
4. Penalize duplicate confidence:

```python
loss_duplicate_conf = -log(1 - pred_conf).mean()
```

Equivalent stable form:

```python
F.binary_cross_entropy(pred_conf.clamp(...), torch.zeros_like(pred_conf))
```

5. If no duplicate unmatched queries exist, return zero.

Add to total:

```python
+ duplicate_conf_weight * losses["loss_duplicate_conf"]
```

### Metrics To Add

- `loss_duplicate_conf`
- `metric_duplicate_unmatched_count`
- `metric_duplicate_unmatched_conf_mean`

### Important Details

- This loss should only inspect unmatched queries that overlap GT strongly.
- This is more targeted than treating all unmatched queries equally.
- It directly addresses bbox-refine failure where many query boxes became plausible and high confidence.

## Task 3: Optional Vertices-Based Projected BBox

### File

`vggt_omega/training/hungarian_losses.py`

### Current Behavior

Projected bbox loss decodes SMPL and builds bbox from first 24 joints:

```python
_, joints = smpl(...)
joints_cam = joints[:, :24] + transl_cam
projected_boxes = points_to_bbox(joints_cam)
```

This is cheap but joints do not cover body silhouette. It is weak as bbox supervision.

### New Config Field

```yaml
loss:
  projected_bbox_source: joints    # joints | vertices
```

Default should be `joints` to preserve current behavior.

### Behavior

If `projected_bbox_source == "joints"`:

- keep current behavior.

If `projected_bbox_source == "vertices"`:

- use SMPL vertices returned from `SMPLLayer`.
- add `pred_transl_cam` to vertices.
- project vertices and build normalized bbox.

Pseudo:

```python
vertices, joints = smpl(...)
points = joints[:, :24] if source == "joints" else vertices
points_cam = points.to(dtype) + transl_cam[:, None, :]
```

### Training Recommendation

For the immediate next run, do not make projected bbox dominant.

Use either:

```yaml
projected_bbox_source: joints
projected_bbox_weight: 0.25
projected_giou_weight: 0.25
```

or, if memory is acceptable:

```yaml
projected_bbox_source: vertices
projected_bbox_weight: 0.5
projected_giou_weight: 0.5
```

Because vertices may increase memory, the first implementation should be config-gated and still decode only matched queries.

## Task 5: Optional Aux BBox/Conf Loss Planning

Do not implement aux loss in the first patch unless the above changes are stable.

Reason:

- Current `SMPLRegressionHead` supports `return_aux`, but `VGGTOmega` does not expose a config flag for it yet.
- Adding aux loss changes model output and loss recursion.
- It is useful, but should come after quality-aware confidence + duplicate suppression are evaluated.

Plan for later:

- Add `model.smpl_return_aux` -> pass to `VGGTOmega` -> `AggregatorSMPLHead(return_aux=True)`.
- Add `loss.aux_weight`.
- Apply only conf/bbox/GIoU loss to aux outputs initially.

## Success Criteria

Relative to Stage 2 projected bbox:

- Keep or improve `mean_best_gt_iou` above `0.63`.
- Improve `box_precision_iou50_conf030` above `0.35` if possible.
- Keep `duplicate_predictions_per_gt_conf030` low, ideally below `0.08`.
- Keep `num_pred_conf030` far below the failed bbox-refine value `7546`; target hundreds to low thousands on the 400-frame subset.
- `metric_conf_pos_mean` should rise above `metric_conf_neg_mean`, but not by making all queries high confidence.
- `metric_duplicate_unmatched_count` should decrease over training.

If NMS metrics are much better than raw metrics, next work should focus on query/conf duplicate suppression.
If NMS metrics are also poor, next work should focus on bbox head/refinement structure.

## Risks

- `matched_iou` confidence target may lower positive confidence if IoUs are low early. Starting from Stage 2 checkpoint mitigates this.
- `duplicate_conf_weight` too high may suppress recall. Start at `1.0`; reduce to `0.5` if recall drops.
- Vertices projected bbox may add memory cost. Keep optional and start with `joints` unless memory is confirmed.
- The bbox sidecar source may not exactly match projected SMPL geometry. This is why projected bbox remains low weight.
- There is still no held-out Test split. Continue to treat metrics as training-subset diagnostics, not final generalization proof.
